# Hostinger Node.js Deployment Guide

## Prerequisites Checklist ✅

Your project is **ready for Hostinger** if:
- ✅ `package.json` is at the **root** of your repository
- ✅ `"start"` script uses `next start -p $PORT` (respects Hostinger's PORT)
- ✅ `"build"` script is defined as `next build`
- ✅ `.env` file is in `.gitignore` (not pushed to GitHub)
- ✅ All dependencies are listed in `package.json`

## Project Structure Requirements

```
project-root/
├── package.json          ← MUST be at root level
├── package-lock.json     ← Generated by npm install
├── next.config.js        ← Next.js configuration
├── tsconfig.json         ← TypeScript configuration
├── .gitignore            ← Contains .env and .next/
├── src/                  ← Source code
├── public/               ← Static files
└── .next/                ← Build output (auto-generated, in .gitignore)
```

## Step-by-Step Hostinger Deployment

### 1. Prepare Your GitHub Repository

**Already Done:**
- ✅ Repository synced with GitHub
- ✅ `.env` is in `.gitignore` (not committed)
- ✅ `.next/` build folder is in `.gitignore`

**Verify on GitHub:**
1. Go to your GitHub repository
2. Check that `package.json` is visible at the root
3. Verify `.env` file is NOT visible (it should be ignored)
4. Verify `.next/` folder is NOT visible

### 2. Create Hostinger Node.js Application

1. **Log in to Hostinger hPanel**
   - Go to https://hpanel.hostinger.com
   - Navigate to your hosting account

2. **Create Node.js Application**
   - Click on **"Node.js"** in the left sidebar
   - Click **"Create Application"**

3. **Configure Application Settings:**

   **Application root:**
   ```
   /
   ```
   *(Leave as root since package.json is at root level)*

   **Application URL:**
   ```
   yourdomain.com
   ```
   *(Or subdomain like app.yourdomain.com)*

   **Application startup file:**
   ```
   npm run start
   ```

   **Node.js version:**
   ```
   18.x or 20.x (Latest LTS)
   ```

   **Application mode:**
   ```
   Production
   ```

### 3. Connect GitHub Repository

1. **In Hostinger Node.js settings:**
   - Click **"Connect to GitHub"**
   - Authorize Hostinger to access your GitHub
   - Select your repository
   - Select the branch (usually `main` or `master`)

2. **Enable Auto-Deploy (Optional but Recommended):**
   - Toggle **"Auto-deploy"** ON
   - Now every push to GitHub will auto-deploy

### 5. Deploy and Build

**Option A: Manual Deploy (First Time)**
1. Click **"Deploy"** or **"Deploy Application"**
2. Hostinger will:
   - Clone your GitHub repository
   - Run `npm install` (install dependencies)
   - Run `npm run build` (build Next.js app)
   - Run `npm run start` (start the server)

**Option B: Auto-Deploy (After Setup)**
1. Push changes to GitHub
2. Hostinger automatically deploys
3. No manual action needed

### 6. Build Process on Hostinger

When you deploy, Hostinger runs these commands:

```bash
# 1. Install dependencies

# 2. Build the application

# 3. Start the application
npm run start
```

**Build Output:**

**Expected build time:** 2-5 minutes

### 8. Common Issues & Solutions

#### ❌ Build Fails: "Cannot find package.json"
**Solution:**
- Verify `package.json` is at the repository root
- Check Application root is set to `/` in Hostinger

#### ❌ Build Fails: "Module not found"
**Solution:**
- Delete `node_modules/` and `package-lock.json` locally
- Run `npm install` to regenerate
- Commit and push to GitHub

#### ❌ Application Shows "502 Bad Gateway"
**Solution:**
- Check if `start` script uses `-p $PORT`:
  ```json
  "start": "next start -p $PORT"
  ```
- Restart application in Hostinger

#### ❌ Environment Variables Not Working
**Solution:**
- Verify variables are added in Hostinger panel
- Variables must start with `NEXT_PUBLIC_` to be accessible in browser
- Restart application after adding variables

#### ❌ Admin Login Not Working
**Solution:**
- Create admin user first (see SECURITY_AND_SETUP.md)
- Check Supabase environment variables are correct
- Verify database is accessible

### 12. Important Hostinger-Specific Notes

**Port Assignment:**
- Hostinger assigns a port automatically
- Your start script MUST use `$PORT` variable:
  ```json
  "start": "next start -p $PORT"
  ```
- ✅ Already configured correctly in your package.json

**File Permissions:**
- No special permissions needed
- Hostinger handles everything automatically

**Database Connection:**
- Uses Supabase (external)
- No local database needed
- Connection via environment variables

**Static Files:**
- Put in `/public` folder
- Automatically served by Next.js
- Example: `/public/logo.png` → `yourdomain.com/logo.png`

**Build Caching:**
- Hostinger caches `node_modules/`
- Speeds up subsequent deploys
- Clear cache if having issues

### 13. Performance Optimization

**Recommended Settings:**

1. **Enable Caching:**
   - Static assets cached automatically
   - Configure in `next.config.js` if needed

2. **Enable Compression:**
   - Already handled by Next.js production build

3. **Optimize Images:**
   - Use Next.js Image component
   - Automatic image optimization

4. **Monitor Bundle Size:**
   ```bash
   npm run build
   # Check output for bundle sizes
   ```

### 14. Backup and Recovery

**Backup Strategy:**
1. **GitHub = Your Backup:**
   - All code is in GitHub
   - Can redeploy anytime

2. **Database Backup:**
   - Supabase auto-backups daily
   - Download backups from Supabase dashboard

3. **Environment Variables:**
   - Keep a secure copy of `.env` locally
   - Document in password manager

**Recovery:**
1. If site goes down, redeploy from GitHub
2. If data lost, restore from Supabase backup
3. If variables lost, re-add from secure copy

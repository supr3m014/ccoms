# WebDev Multi‑Agent Template (General)

## Purpose
A general, project‑agnostic **website/app development** doc system for staged delivery, proof gates, no drift, and transparent reporting.

## Mandatory reading order (do not skip)
1) `docs/AGENT.md`
2) `docs/00_GOVERNANCE.md`
3) `docs/AGENTS.md`
4) `docs/RUNBOOK.md`
5) `docs/VERIFY.md`
6) `docs/PROMPTS.md`
7) `docs/LEARNED.md`
8) `docs/REPO_MAP.md` (generated in Stage 00)

## Definition of Done
A stage is DONE only when:
- its Stage Proof passes (see `docs/RUNBOOK.md` + `docs/VERIFY.md`), AND
- QA/FA records pass/fail evidence, AND
- a Stage Report is written (`docs/REPORTING.md`).

## Reports
Stage reports live in `docs/reports/`.
